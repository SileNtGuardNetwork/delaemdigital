# Delaem Digital — Website Factory OS

Production repository for the Delaem Digital Website Factory: an AI-first system for building premium ClientFlow websites.

## Mission

This repository is not a generic website template.

It is the operational codebase for producing **premium ClientFlow sites**:

```text
Discovery -> Blueprint -> Copy -> Design -> Next.js -> Integrations -> Analytics -> QA Gate -> Launch -> Case
```

Delaem Digital does not sell isolated websites, bots, or advertising as disconnected services. The factory builds controlled client acquisition routes: offer, site, form, Telegram/CRM, analytics, AI qualification, and improvement loop.

## Factory Principles

1. Strategy before interface.
2. Offer before visuals.
3. Blueprint before design.
4. Design before code.
5. Code before automation.
6. Analytics before traffic.
7. QA before launch.
8. Evidence after launch.
9. Case after evidence.
10. Reusable asset after every project.

## Forbidden

- Starting with a hero image before strategy.
- Designing without business clarity.
- Fake dashboards.
- Random decorative cards.
- Neon/cyberpunk styling for Delaem Digital premium sites.
- Raster text inside generated images.
- Forms without consent.
- Launch without tested lead flow.
- Traffic without analytics.
- AI actions without logs and approval gates.
- Handoff without documentation.
