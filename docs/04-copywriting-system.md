# Copywriting System

Status: v1 foundation
Purpose: define the writing standard for premium ClientFlow websites.

## 1. Core Principle

Copy is not decoration. Copy is route control.

Every block of text must move the user through the ClientFlow route:

```text
Attention -> Understanding -> Trust -> Action -> Lead -> Processing -> Improvement
```

A website is weak if it sounds premium but does not explain:

1. who this is for
2. what problem is solved
3. what result is expected
4. why this approach is different
5. what the user should do next

## 2. Hero Formula

Hero must answer in 3 to 5 seconds:

1. Who is speaking or who the company is.
2. Who the offer is for.
3. What system or result is being built.
4. Why it is not a generic service.
5. What the next step is.

Structure:

- Kicker: role, company, or category marker.
- H1: main result.
- Subheadline: audience plus mechanism plus differentiation.
- Primary CTA: commercial next step.
- Secondary CTA: softer route.
- Trust marker: proof, method, constraint, or stack.

## 3. Good vs Bad Copy

Bad:

```text
We create modern websites for your business.
```

Better:

```text
We build traffic-ready websites that turn attention into qualified requests and show where the client route loses conversion.
```

Bad:

```text
Leave a request.
```

Better:

```text
Get a system review.
```

Bad:

```text
We guarantee leads.
```

Better:

```text
We build and measure the route that helps traffic become requests. Lead volume depends on offer, market, budget, traffic, and processing quality.
```

## 4. Section Copy Rules

Each section must have:

- one commercial job
- one main idea
- one primary action or logical transition
- no generic filler
- no repeated claims
- no unsupported promises

Section structure:

```text
Eyebrow
Headline
Subheadline
Body or proof structure
CTA
Microcopy if needed
```

## 5. CTA System

CTA must match user readiness.

Primary CTA examples:

- Get a system review
- Discuss the project
- Start with diagnostic
- Build the website system

Secondary CTA examples:

- View the approach
- Go to Telegram
- See cases
- Check the ClientFlow map

Forbidden as main CTA:

- Learn more
- Send
- More details
- Click here
- Submit

## 6. Product Copy

Each product card must explain:

- who it is for
- what problem it solves
- what is included
- what result it creates
- starting price or estimate logic
- next step

Do not make product cards look like cheap tariff cards.

## 7. FAQ Copy

FAQ must close real objections, not fill space.

Required questions for most ClientFlow websites:

1. How is this different from a normal website?
2. What if I already have a website?
3. Can we start small?
4. What do you need from us?
5. How long does the project take?
6. Do you guarantee leads?
7. What happens after launch?
8. Where do requests go?
9. What analytics will be visible?
10. What is not included?

## 8. Risky Claims

Avoid or heavily qualify claims about:

- guaranteed leads
- guaranteed sales
- guaranteed profit
- guaranteed ranking
- medical results
- investment returns
- legal outcomes
- advertising approval
- platform stability

Use factual, controlled language.

## 9. Tone

Default tone:

- calm
- direct
- architectural
- premium
- specific
- commercially mature

Avoid:

- hype
- manipulation
- generic agency phrases
- cheap urgency
- empty innovation language
- overpromising
- excessive slang
