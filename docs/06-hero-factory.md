# Hero Factory

Status: v1 foundation
Purpose: define the production standard for first screens of premium ClientFlow websites.

## 1. Mission

The hero section must explain the offer in 3 to 5 seconds and lead the user to the correct next action.

A hero is not a decoration layer. It is the first routing point of the ClientFlow system.

## 2. Required Questions

Every hero must answer:

1. Who is speaking?
2. Who is this for?
3. What result or system is being built?
4. Why is this different from a generic service?
5. What should the user do next?

## 3. Hero Structure

Required elements:

- logo or brand marker
- navigation if full website
- kicker or category marker
- H1
- subheadline
- primary CTA
- secondary CTA
- trust marker or proof row
- visual anchor

Optional elements:

- founder portrait
- route map
- system diagram
- product proof
- case marker
- stack marker

## 4. H1 Rules

A strong H1 must:

- communicate the main result
- be specific
- avoid abstract agency language
- avoid hype
- be readable on mobile
- work without supporting visual

Weak patterns:

- Modern solutions for your business
- We create websites that sell
- Digital transformation for everyone
- Innovative agency for growth

Strong patterns:

- Build a traffic-ready website that turns attention into qualified requests
- Create a controlled route from first touch to client request
- Turn scattered digital tools into one measurable client acquisition system

## 5. CTA Hierarchy

Primary CTA:

- commercial next step
- diagnostic next step
- project discussion
- system review

Secondary CTA:

- Telegram channel
- view approach
- view cases
- learn the method

Rules:

- do not use more than two main CTAs
- primary CTA must be visually dominant
- secondary CTA must not compete
- CTA must be visible above the fold on desktop and early on mobile

## 6. Visual Patterns

Approved patterns:

### Founder-led Control Room

Use when trust depends on the founder or expert.

Structure:

- copy left
- founder or premium environment right
- compact proof row
- controlled route motif

### ClientFlow Architecture Map

Use when the product is a system.

Structure:

- strong H1
- modular route map
- steps from attention to lead
- clear CTA

### Premium Command Interface

Use when the product involves analytics, automation, AI, CRM, or operational control.

Structure:

- text plus system modules
- real UI labels as HTML
- no fake metrics
- no random dashboard decoration

## 7. Forbidden Patterns

Forbidden:

- generic SaaS right-side dashboard
- random floating cards
- fake analytics charts
- AI raster text
- neon or cyberpunk overload
- visual that competes with headline
- portrait that is too dark to read
- buttons shaped as pills unless design system allows badges only
- CTA below first viewport on desktop

## 8. Desktop Rules

Target first:

- 1440 by 900
- content max width 1280 to 1360px
- header 72 to 84px
- CTA above fold
- H1 large but controlled
- proof row compact
- visual supports offer, not the opposite

## 9. Mobile Rules

Target widths:

- 390px
- 430px

Rules:

- no horizontal overflow
- text first
- H1 readable without awkward wrapping
- CTA visible early
- proof row wraps cleanly
- visual must not kill readability
- cookie banner must not cover primary CTA

## 10. Motion Rules

Allowed:

- subtle reveal
- CTA hover
- route-line motion
- small system state transitions

Forbidden:

- heavy entrance animations
- endless distracting loops
- animation that delays CTA
- scroll hijacking
- motion without prefers-reduced-motion support

## 11. Hero Production Process

Step 1: Read discovery and blueprint.
Step 2: Define the commercial job of the hero.
Step 3: Create 3 concepts.
Step 4: Select one winner.
Step 5: Build desktop layout.
Step 6: Build mobile layout.
Step 7: Run visual QA.
Step 8: Run copy QA.
Step 9: Run conversion QA.
