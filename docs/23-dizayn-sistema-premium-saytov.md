# Дизайн-система premium-сайтов

Статус: v1 foundation
Назначение: зафиксировать визуальный стандарт сайтов Делаем Диджитал до начала кода и дизайна.

## 1. Главная позиция

Premium-дизайн — это не тёмный фон, не карточки и не красивые градиенты.

Premium-дизайн в фабрике — это:

- ясная иерархия;
- дорогая визуальная тишина;
- сильная типографика;
- контролируемый motion;
- точные отступы;
- понятные CTA;
- минимум визуального шума;
- ощущение системы;
- скорость и комфорт;
- доверие.

Сайт должен выглядеть как управляемая digital-система, а не как случайная подборка блоков.

## 2. Базовый визуальный код

Основной стиль:

```text
тёмный premium minimalism
graphite / obsidian surfaces
copper action accents
steel-blue system accents
soft glow
thin borders
clean typography
large calm spacing
```

Ощущение:

```text
дорого
спокойно
технологично
собранно
не кричаще
не инфобизнес
не крипто-неон
не шаблон SaaS
```

## 3. Цветовая система

### База

```text
graphite black
obsidian dark
deep charcoal
soft surface black
```

### Текст

```text
primary white
secondary cool gray
muted steel gray
```

### Акценты

```text
copper — действия, CTA, важные маркеры
steel-blue — система, диагностика, технологичность
soft green — успех, подтверждение
soft red — ошибка, риск
```

## 4. Правила цвета

- Copper не использовать как массовый декоративный цвет.
- Steel-blue использовать для маршрута, системы, аналитики, диагностики.
- Green только для success state.
- Red только для ошибок и рисков.
- Не использовать кислотные цвета.
- Не делать rainbow gradients.
- Не делать neon/cyberpunk.
- Не делать дешёвую “AI-glow” эстетику.

## 5. Типографика

Базовая пара:

- Onest — интерфейс, UI, секции;
- Manrope — сильные заголовки, брендовые акценты.

Правила:

- H1 должен быть сильным, но читаемым;
- не делать слишком тонкий текст;
- не делать мелкий серый текст на тёмном фоне;
- line-height должен давать воздух;
- mobile H1 не должен ломаться на странные строки;
- пользователь должен понять первый экран за 3-5 секунд.

## 6. Сетка и отступы

Desktop:

```text
max width: 1280-1360px
section padding: 96-140px
hero min-height: 760-900px
container side padding: 48-64px
```

Tablet:

```text
section padding: 72-96px
container side padding: 32px
```

Mobile:

```text
section padding: 56-80px
container side padding: 20-24px
CTA early and visible
no horizontal overflow
```

## 7. Карточки и панели

Карточки должны выглядеть как элементы системы, а не как random cards.

Разрешено:

- тонкая граница;
- тёмная glass-поверхность;
- soft blur;
- subtle shadow;
- glow только как акцент;
- чёткая иерархия внутри карточки.

Запрещено:

- одинаковая сетка карточек в каждой секции;
- жирные градиенты на всех панелях;
- дешёвый glassmorphism;
- слишком много shadows;
- random floating cards;
- fake metrics;
- fake dashboards.

## 8. Кнопки

PrimaryButton:

- copper background;
- не pill по умолчанию;
- radius 10-14px;
- уверенный hover;
- visible focus;
- clear label;
- не использовать “Подробнее” как primary.

SecondaryButton:

- border или transparent surface;
- слабее primary;
- не конкурирует;
- подходит для Telegram, кейсов, подхода.

Запрещено:

- слишком много CTA в одном экране;
- одинаковая сила всех кнопок;
- мелкие кнопки на mobile;
- кнопки без analytics event.

## 9. Первый экран

Hero должен иметь один доминирующий смысл.

Допустимые паттерны:

1. Founder-led hero.
2. System map hero.
3. Diagnostic hero.
4. Productized service hero.

Обязательное:

- оффер виден сразу;
- CTA выше fold;
- визуал поддерживает смысл;
- нет конкуренции между портретом и текстом;
- mobile version сохраняет смысл;
- secondary CTA ведёт в лид-магнит или Telegram-бот.

## 10. Визуальный ритм страницы

Страница не должна быть набором одинаковых блоков.

Рекомендуемый ритм:

```text
Hero
-> Problem Map
-> ClientFlow System Map
-> Product Composition
-> AI Funnel
-> Before / After
-> Process
-> Offer Cards
-> Trust / Proof
-> Founder
-> Lead Magnet
-> FAQ
-> Final CTA
```

Визуальные паттерны должны чередоваться:

- split layout;
- route map;
- comparison;
- timeline;
- cards;
- editorial block;
- diagnostic panel;
- form panel.

## 11. Изображения и портреты

Для founder-led сайтов:

- портрет должен усиливать доверие;
- лицо не искажать;
- не использовать stock people;
- не заменять владельца случайным AI-персонажем;
- фон должен соответствовать premium-настроению;
- left/right composition должна оставлять место для текста и CTA.

Для клиентских сайтов:

- реальные фото лучше fake visuals;
- если фото плохие, использовать системные визуальные метафоры;
- не использовать изображения с текстом внутри;
- все важные слова должны быть HTML-текстом.

## 12. ИИ-автоворонка в дизайне

ИИ-автоворонка должна выглядеть не как скучный bot link, а как ценный диагностический продукт.

Дизайн-блок должен показывать:

- что пользователь получит;
- как быстро;
- почему это полезно;
- что будет после перехода в Telegram;
- как ИИ помогает;
- какой лид-магнит выдаётся;
- какой следующий шаг.

Разрешённые визуальные паттерны:

- diagnostic card;
- Telegram flow preview;
- step-by-step chain;
- lead magnet preview;
- scorecard preview;
- owner lead-card preview without fake personal data.

## 13. Формы

Форма должна выглядеть как доверительный рабочий инструмент.

Требования:

- labels видимы;
- ошибки понятны;
- consent не спрятан;
- submit label конкретный;
- success state приятный;
- failure state спокойный;
- mobile input удобный;
- поля не перегружены.

## 14. Mobile-first качество

Mobile version не должна быть обрезанной desktop-версией.

Проверить:

- H1 читается;
- CTA виден рано;
- visual не мешает;
- карточки не превращаются в бесконечную кашу;
- форма удобна;
- Telegram CTA очевиден;
- sticky/header не закрывает контент;
- cookie banner не блокирует главное действие.

## 15. Запреты

Нельзя:

- generic SaaS landing;
- neon/cyberpunk;
- fake dashboard;
- random floating cards;
- stock people;
- raster text;
- перегрузка blur/glow;
- однотипные карточки весь сайт;
- хаотичный motion;
- мелкая типографика;
- плохой контраст;
- CTA ниже первого экрана;
- дизайн без связи с оффером.

## 16. Дизайн Gate

Оценить по 10-балльной шкале.

| Блок | Оценка | Комментарий |
| --- | ---: | --- |
| Ясность первого экрана |  |  |
| Premium-восприятие |  |  |
| Визуальная иерархия |  |  |
| Типографика |  |  |
| Цвет и акценты |  |  |
| CTA |  |  |
| Mobile |  |  |
| ИИ-автоворонка |  |  |
| Формы |  |  |
| Скорость и лёгкость |  |  |

Итого: /100

Статусы:

```text
90-100: дизайн готов к реализации
75-89: условно готов, нужны правки
ниже 75: не готов
```

Критичные блокеры:

- непонятный первый экран;
- CTA не виден;
- mobile неудобен;
- форма выглядит недоверительно;
- ИИ-автоворонка не воспринимается как ценность;
- сайт выглядит шаблонно;
- visual noise мешает смыслу.

## 17. Решение

Дизайн фабрики должен быть системным.

Каждый сайт может отличаться под нишу, но базовое качество должно сохраняться:

```text
ясность -> доверие -> премиальность -> удобство -> скорость -> действие
```
