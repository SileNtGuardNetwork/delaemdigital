# Стандарт motion и premium-эффектов

Статус: v1 foundation
Назначение: зафиксировать правила анимаций и визуальных эффектов для premium ClientFlow-сайтов.

## 1. Главная позиция

Motion нужен не для вау-шума.

Motion в фабрике нужен для:

- усиления смысла;
- направления внимания;
- объяснения системы;
- повышения ощущения качества;
- улучшения обратной связи;
- комфортного взаимодействия;
- премиального восприятия.

Если анимация не помогает понять, доверять или действовать — она не нужна.

## 2. Базовый характер motion

Стиль motion:

```text
спокойный
точный
дорогой
не резкий
не игровой
не рекламно-кричащий
не перегруженный
```

Ощущение:

```text
система оживает
интерфейс отвечает
маршрут становится понятнее
пользователь чувствует контроль
```

## 3. Разрешённые типы motion

### Reveal-анимации

Использовать для:

- появления секций;
- заголовков;
- карточек;
- CTA;
- proof-блоков.

Правила:

- коротко;
- мягко;
- без задержек, мешающих чтению;
- не анимировать каждый мелкий элемент;
- не превращать страницу в презентацию.

### Hover-анимации

Использовать для:

- кнопок;
- карточек;
- ссылок;
- интерактивных схем.

Правила:

- движение 1-3px;
- мягкая смена border / glow / background;
- без прыжков;
- без агрессивного scale.

### Системные анимации

Использовать для:

- ClientFlow-карты;
- маршрута заявки;
- Telegram-flow preview;
- ИИ-автоворонки;
- progress states;
- lead score preview.

Правила:

- показывать связь этапов;
- помогать понять процесс;
- не использовать fake metrics;
- не имитировать реальные данные без маркировки.

### Form feedback

Обязательно:

- loading state;
- success state;
- failure state;
- validation feedback;
- disabled state.

Motion должен помогать пользователю понять, что происходит.

## 4. Premium-эффекты

Разрешённые эффекты:

```text
soft glow
thin light line
subtle gradient border
glass surface
rim light
noise texture
slow ambient background
route-line animation
```

Использовать дозированно.

Один экран не должен содержать все эффекты одновременно.

## 5. Запрещённые эффекты

Нельзя:

- heavy parallax;
- scroll hijacking;
- бесконечные агрессивные loop-анимации;
- кислотный neon;
- cyberpunk-overload;
- random floating cards;
- fake dashboard animation;
- анимированный текст, который ухудшает чтение;
- blur/glow на всём подряд;
- scale 1.1+ на hover для карточек;
- motion, который ломает mobile.

## 6. Motion по секциям

### Первый экран

Допустимо:

- мягкий fade-up заголовка;
- спокойное появление CTA;
- лёгкое оживление route map;
- слабый ambient light.

Запрещено:

- задерживать появление оффера;
- скрывать CTA за анимацией;
- делать портрет слишком активным;
- перегружать hero частицами.

### Карта проблемы

Допустимо:

- последовательное раскрытие симптомов;
- подсветка слабых мест;
- лёгкая линия маршрута.

### ClientFlow System

Допустимо:

- route-line animation;
- step highlight;
- state transition;
- hover по этапам.

### ИИ-автоворонка

Допустимо:

- preview цепочки сообщений;
- шаги Telegram-бота;
- выдача лид-магнита;
- прогресс диагностики;
- lead score as illustrative UI.

Важно:

Не показывать fake personal data.

### Форма

Обязательно:

- loading;
- success;
- failure;
- validation.

## 7. Performance budget

Motion не должен убивать скорость.

Правила:

- предпочитать transform и opacity;
- избегать анимации width, height, top, left;
- не использовать тяжёлые canvas/WebGL без причины;
- не подключать большие библиотеки ради одного эффекта;
- lazy-load тяжёлые интерактивы;
- тестировать mobile.

## 8. Accessibility

Обязательно учитывать:

```text
prefers-reduced-motion
```

Если пользователь отключил motion:

- убрать сложные переходы;
- оставить понятные state changes;
- не ломать функциональность;
- не скрывать контент.

## 9. Библиотеки

Допустимо:

- CSS transitions;
- Framer Motion, если justified;
- native IntersectionObserver;
- lightweight utilities.

Не использовать без отдельного решения:

- тяжёлые animation frameworks;
- full WebGL scenes;
- Three.js ради декора;
- scroll libraries, которые перехватывают поведение страницы.

## 10. Motion tokens

Нужны базовые токены:

```text
duration-fast: 120-180ms
duration-base: 220-320ms
duration-slow: 420-700ms
ease-standard
ease-out-premium
ease-in-out-soft
hover-lift: 1-3px
reveal-distance: 12-24px
```

## 11. QA motion

Проверить:

- motion не мешает понять оффер;
- CTA появляется быстро;
- mobile не тормозит;
- нет layout shift;
- форма даёт понятную обратную связь;
- `prefers-reduced-motion` учтён;
- эффекты не выглядят дешево;
- нет визуального шума;
- сайт ощущается быстрым.

## 12. Motion Gate

Оценить по 10-балльной шкале.

| Блок | Оценка | Комментарий |
| --- | ---: | --- |
| Поддержка смысла |  |  |
| Premium-восприятие |  |  |
| Скорость |  |  |
| Mobile |  |  |
| CTA visibility |  |  |
| Form feedback |  |  |
| Accessibility |  |  |
| Отсутствие шума |  |  |
| Уместность эффектов |  |  |
| Техническая реализация |  |  |

Итого: /100

Статусы:

```text
90-100: motion готов
75-89: условно готов
ниже 75: не готов
```

Критичные блокеры:

- motion скрывает оффер;
- CTA появляется слишком поздно;
- mobile тормозит;
- есть scroll hijacking;
- нет reduced-motion fallback;
- эффекты выглядят дешево;
- анимация ломает форму или секции.

## 13. Решение

Motion должен быть частью качества, а не украшением.

Правильная формула:

```text
смысл -> внимание -> доверие -> комфорт -> действие
```

Если motion не помогает этой цепочке, он удаляется.
