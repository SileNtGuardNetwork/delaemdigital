# SEO-система под ключ

Статус: v1 foundation
Назначение: зафиксировать SEO-стандарт для сайтов Делаем Диджитал до дизайна, кода и запуска.

## 1. Главная позиция

SEO нельзя добавлять в конце.

SEO влияет на:

- структуру сайта;
- структуру страниц;
- заголовки;
- copy-doc;
- FAQ;
- скорость;
- изображения;
- schema.org;
- internal linking;
- техническую архитектуру;
- контентный план.

Сайт, который продаётся как “SEO под ключ”, должен иметь SEO-логику уже на этапе конверсионного прототипа.

## 2. Что значит SEO под ключ

Минимальный состав:

```text
1. SEO-диагностика ниши
2. Поисковые интенты
3. Семантическая карта
4. Sitemap
5. Структура страниц
6. Title / Description
7. H1-H3 структура
8. SEO-copy requirements
9. FAQ для SEO
10. Schema.org
11. OpenGraph
12. Robots
13. Sitemap.xml
14. Canonical
15. Image alt strategy
16. Internal linking
17. Core Web Vitals
18. SEO Gate перед запуском
```

## 3. SEO-диагностика ниши

Перед прототипом определить:

- что ищут клиенты;
- какие запросы коммерческие;
- какие запросы информационные;
- какие запросы локальные;
- какие запросы лучше вести на главную;
- какие запросы требуют отдельных страниц;
- какие запросы нецелевые;
- где есть высокая конкуренция;
- где есть низкочастотные возможности.

## 4. Поисковые интенты

Типы интентов:

```text
коммерческий
информационный
локальный
брендовый
сравнительный
проблемный
транзакционный
```

Каждая страница должна иметь один главный интент.

Запрещено смешивать на одной странице слишком много разных интентов.

## 5. Семантическая карта

Для проекта создать таблицу:

| Страница | Главный интент | Основные фразы | Вторичные фразы | CTA | Статус |
| --- | --- | --- | --- | --- | --- |
| Главная |  |  |  |  |  |
| Продукт |  |  |  |  |  |
| Ниша |  |  |  |  |  |
| Кейсы |  |  |  |  |  |
| Блог / статьи |  |  |  |  |  |

## 6. Структура страниц

Базовая структура для сайта услуг:

```text
/
/products или /uslugi
/products/site
/products/traffic
/products/system
/cases
/cases/[slug]
/about
/contact
/privacy
/consent
/cookies
/terms
```

Для SEO-проектов добавить:

```text
/nishi/[slug]
/services/[slug]
/blog/[slug]
/guides/[slug]
```

Не создавать лишние страницы без интента.

## 7. Title и Description

### Title

Правила:

- главный интент ближе к началу;
- бренд в конце;
- без переспама;
- 45-65 символов как ориентир;
- не обещать гарантированные результаты.

Формат:

```text
[Услуга / результат] для [аудитория / ниша] — [бренд]
```

### Description

Правила:

- объяснить пользу;
- указать отличие;
- добавить CTA;
- 130-160 символов как ориентир;
- без кликбейта.

## 8. H1-H3 структура

Правила:

- один H1 на страницу;
- H1 должен поддерживать главный интент;
- H2 раскрывают структуру;
- H3 используются для деталей;
- не делать заголовки только ради дизайна;
- не использовать H-теги как декоративный текст.

## 9. SEO-copy

SEO-copy не должен ломать конверсию.

Правила:

- сначала смысл и конверсия;
- потом ключевые фразы;
- не переспамливать;
- не писать “простыню” ради SEO;
- FAQ использовать для возражений и long-tail;
- ключи должны быть естественными;
- коммерческие блоки не превращать в статью.

## 10. FAQ для SEO

FAQ должен закрывать:

- реальные возражения;
- long-tail запросы;
- вопросы по процессу;
- вопросы по срокам;
- вопросы по цене;
- вопросы по гарантиям;
- вопросы по ИИ-автоворонке;
- вопросы по трафику и аналитике.

FAQ можно разметить schema.org, если это уместно.

## 11. Schema.org

Рекомендуемые типы:

```text
Organization
LocalBusiness
ProfessionalService
Service
Product
FAQPage
BreadcrumbList
Article
Review / AggregateRating только при наличии реальных оснований
```

Запрещено:

- fake reviews;
- fake ratings;
- schema, не соответствующая контенту страницы;
- разметка услуг, которых нет на странице.

## 12. OpenGraph

Каждая важная страница должна иметь:

- og:title;
- og:description;
- og:image;
- og:type;
- canonical URL.

OG image должен быть читаемым, но важный SEO-текст не должен жить только в изображении.

## 13. Sitemap и robots

Обязательно:

```text
/sitemap.xml
/robots.txt
canonical URLs
noindex для технических страниц
```

В sitemap включать только нужные индексируемые страницы.

Не включать:

- черновики;
- тестовые страницы;
- приватные страницы;
- thank-you страницы, если они не нужны для индексации.

## 14. Изображения

Правила:

- использовать alt там, где изображение несёт смысл;
- декоративные изображения могут иметь пустой alt;
- не использовать огромные изображения без оптимизации;
- WebP/AVIF где уместно;
- priority только для hero-critical images;
- lazy loading для вторичных изображений;
- не использовать изображение вместо текста.

## 15. Internal linking

Сайт должен помогать пользователю и поисковику понять структуру.

Минимум:

- главная ведёт на продукты;
- продукты ведут на диагностику;
- кейсы ведут на похожие продукты;
- статьи ведут на продукт;
- footer содержит legal и ключевые страницы;
- breadcrumbs для вложенных страниц.

## 16. Local SEO

Для локальных бизнесов:

- город/регион в нужных местах;
- контакты;
- карта, если нужно;
- локальные страницы;
- schema LocalBusiness;
- часы работы, если применимо;
- отзывы только реальные.

## 17. SEO и ИИ-автоворонка

SEO-страницы должны вести не только в заявку, но и в лид-магнит.

Примеры:

```text
SEO-страница про сайт для клиники
-> CTA: Получить чек-лист доверия пациента
-> Telegram-бот
-> диагностика
-> трипвайер: мини-аудит сайта клиники
```

SEO-контент должен подводить к ИИ-воронке, а не жить отдельно.

## 18. Техническое SEO

Проверить:

- правильный lang;
- metadata;
- canonical;
- robots;
- sitemap;
- no broken links;
- no duplicate titles;
- no duplicate H1;
- structured data valid;
- responsive layout;
- Core Web Vitals;
- image optimization;
- font loading;
- no hidden critical content;
- no client-only critical text.

## 19. SEO Gate

Оценить по 10-балльной шкале.

| Блок | Оценка | Комментарий |
| --- | ---: | --- |
| Интент |  |  |
| Семантическая карта |  |  |
| Sitemap |  |  |
| Title / Description |  |  |
| H-структура |  |  |
| SEO-copy |  |  |
| Schema.org |  |  |
| Internal linking |  |  |
| Техническое SEO |  |  |
| Связь с конверсией |  |  |

Итого: /100

Статусы:

```text
90-100: SEO готово
75-89: условно готово
ниже 75: не готово
```

Критичные блокеры:

- нет главного интента;
- нет title/description;
- нет H1;
- нет sitemap для SEO-проекта;
- нет robots;
- есть дубль H1;
- важный текст только в изображении;
- SEO разрушает конверсию;
- нет связи SEO-страниц с CTA и лид-магнитом.

## 20. Роль GPT Business

GPT Business отвечает за:

- SEO-логику на этапе прототипа;
- структуру страниц;
- title/description;
- H-структуру;
- FAQ;
- связь SEO с ИИ-автоворонкой;
- SEO Gate.

## 21. Роль Cursor Pro+

Cursor отвечает за:

- metadata implementation;
- sitemap.xml;
- robots.txt;
- canonical;
- schema.org;
- breadcrumbs;
- image optimization;
- performance-related SEO;
- technical checks.

Cursor не должен сам менять SEO-стратегию без задания от GPT Business.

## 22. Решение

SEO входит в фабрику не как отдельная услуга после сайта, а как обязательный слой:

```text
интент -> структура -> copy -> schema -> скорость -> CTA -> лид-магнит -> аналитика
```

Сайт считается SEO-ready только после прохождения SEO Gate.
