# Стандарт ИИ-автоворонки

Статус: v1 foundation
Назначение: зафиксировать обязательный продуктовый стандарт ИИ-автоворонки для сайтов, собранных по технологии ClientFlow System.

## 1. Главная позиция

ИИ-автоворонка — это не просто форма на сайте и не обычный Telegram-бот.

ИИ-автоворонка — это под ключ упакованный дизайнерский современный Telegram-бот, связанный с сайтом, лид-магнитом, welcome-цепочкой, ИИ-ядром, прогревом, сегментацией и монетизацией.

Базовая формула:

```text
Сайт -> CTA -> Telegram-бот -> лид-магнит -> welcome-цепочка -> ИИ-сбор данных -> прогрев -> трипвайер -> оплата -> следующий шаг
```

## 2. Роль в продукте

Каждый серьёзный сайт фабрики должен рассматриваться не как отдельная страница, а как вход в ClientFlow System.

ИИ-автоворонка отвечает за:

1. Удержание пользователя после первого интереса.
2. Выдачу ценности до продажи.
3. Сбор данных о потребности пользователя.
4. Сегментацию.
5. Прогрев.
6. Продажу трипвайера.
7. Подготовку к основному продукту.
8. Передачу владельцу качественной карточки лида.
9. Аналитику поведения пользователя.

## 3. Состав ИИ-автоворонки

Минимальный состав:

```text
1. Точка входа на сайте
2. Telegram-бот
3. Лид-магнит
4. Welcome-цепочка
5. ИИ-ядро
6. Сбор данных
7. Сегментация
8. Прогрев
9. Трипвайер
10. Приём оплаты
11. Карточка лида владельцу
12. Аналитика
13. Follow-up
```

## 4. Точка входа на сайте

На сайте должен быть один или несколько входов в ИИ-автоворонку.

Типы входов:

- кнопка в hero;
- блок диагностики;
- финальный CTA;
- popover после просмотра ключевой секции;
- отдельная страница `/diagnostic`;
- QR-код для офлайн-точек;
- Telegram deep-link с параметрами.

Правило:

CTA в ИИ-воронку должен обещать конкретную пользу, а не просто “перейти в бот”.

Плохо:

```text
Перейти в Telegram
```

Хорошо:

```text
Получить карту потерь заявок
Пройти мини-аудит сайта
Получить план улучшения воронки
Получить диагностический чек-лист
```

## 5. Лид-магнит

Лид-магнит — это первая ценность, которую пользователь получает через бота.

Форматы:

- PDF-чек-лист;
- персональный мини-аудит;
- интерактивная диагностика;
- карта потерь;
- калькулятор;
- подбор решения;
- мини-курс;
- серия сообщений;
- видео-разбор;
- таблица;
- шаблон;
- scorecard.

Требования:

- должен быть связан с болью пользователя;
- должен подводить к продукту;
- не должен быть пустым “скачайте PDF”;
- должен давать быстрый эффект;
- должен собирать данные для дальнейшей сегментации;
- должен иметь измеримое событие выдачи.

## 6. Welcome-цепочка

Welcome-цепочка знакомит пользователя с бизнесом, экспертом или компанией.

Минимальная структура:

```text
Сообщение 1: выдача лид-магнита
Сообщение 2: кто мы и чем полезны
Сообщение 3: проблема пользователя и типичная ошибка
Сообщение 4: метод / ClientFlow System / подход
Сообщение 5: кейс или доказательство
Сообщение 6: предложение трипвайера
Сообщение 7: мягкий follow-up
```

Правила:

- сообщения короткие;
- один смысл на одно сообщение;
- без агрессивных продаж;
- каждая цепочка учитывает нишу;
- пользователь может выбрать интерес;
- есть путь к человеку;
- есть отписка или пауза.

## 7. ИИ-ядро

ИИ-ядро отвечает за сбор информации, анализ и дальнейший персонализированный прогрев.

Функции:

1. Задаёт уточняющие вопросы.
2. Классифицирует пользователя.
3. Определяет боль.
4. Определяет стадию готовности.
5. Определяет подходящий продукт.
6. Считает lead score.
7. Выбирает ветку прогрева.
8. Формирует карточку лида владельцу.
9. Предлагает следующий шаг.
10. Помогает продать трипвайер.

Запрещено:

- обещать гарантированный результат;
- давать медицинские, юридические, финансовые выводы как окончательное заключение;
- принимать критичные решения без человека;
- скрывать, что пользователь общается с автоматизированной системой, если это важно юридически;
- собирать лишние персональные данные.

## 8. Сбор данных

ИИ-воронка должна собирать не просто контакт, а контекст.

Минимальные данные:

```text
source
utm
entry_point
lead_magnet
answers
pain
niche
budget_range
urgency
readiness_stage
interest
contact
consent
```

Возможные стадии готовности:

```text
cold
problem_aware
solution_aware
vendor_aware
ready_to_buy
```

## 9. Сегментация

Сегментация нужна для персонального прогрева.

Примеры сегментов:

- только изучает проблему;
- есть сайт, но нет заявок;
- есть трафик, но заявки слабые;
- хочет запустить новый продукт;
- готов к проекту;
- низкий бюджет;
- нецелевой;
- нужен быстрый аудит;
- нужен основной продукт.

Каждый сегмент получает свою ветку.

## 10. Прогрев

Прогрев строится вокруг пользы, доверия и следующего шага.

Типы сообщений:

- образовательное;
- диагностическое;
- кейсовое;
- сравнительное;
- разбор ошибки;
- демонстрация метода;
- предложение трипвайера;
- предложение основного продукта;
- возврат к диалогу.

Правила:

- не спамить;
- соблюдать частоту;
- уважать паузы;
- не давить;
- логировать взаимодействия;
- не смешивать все продукты в одну цепочку.

## 11. Трипвайер

ИИ-автоворонка должна уметь продавать недорогой входной продукт на 100% автоматически.

Трипвайер зависит от ниши.

Возможные трипвайеры:

- экспресс-аудит сайта;
- мини-разбор воронки;
- платный диагностический отчёт;
- шаблон;
- консультация;
- мини-курс;
- чек-лист с разбором;
- быстрый прототип;
- нишевой аудит.

Требования:

- конкретный результат;
- низкий барьер оплаты;
- понятный срок выдачи;
- простая оплата;
- автоматическая выдача;
- путь к основному продукту.

## 12. Оплата

Платёжный контур может быть подключён в v2/v3, но должен быть предусмотрен архитектурно.

Возможные варианты:

- ЮKassa;
- CloudPayments;
- Robokassa;
- Tinkoff/Т-Банк;
- Telegram payments;
- invoice-link;
- ручной счёт для B2B.

Правила:

- не хранить платёжные данные;
- использовать платёжного провайдера;
- фиксировать payment status;
- после оплаты выдавать продукт или следующий шаг;
- уведомлять владельца;
- отправлять пользователю подтверждение.

Статусы оплаты:

```text
payment_started
payment_pending
payment_success
payment_failed
payment_refunded
```

## 13. Карточка лида владельцу

После диагностики владелец должен получить не просто контакт, а понятную карточку.

Карточка включает:


## 15. Нишевые лид-магниты

### Эксперт

- аудит упаковки;
- карта доверия;
- план улучшения личного сайта;
- мини-разбор воронки консультаций.

### B2B-услуги

- карта потерь заявок;
- аудит маршрута клиента;
- checklist сайта под трафик;
- калькулятор слабых мест.

### Медицина

- аудит доверия сайта;
- чек-лист записи пациента;
- диагностика пути пациента;
- разбор карточки услуги.

Ограничение: без медицинских диагнозов и обещаний результата.

### Недвижимость

- чек-лист упаковки объекта;
- аудит заявки;
- калькулятор готовности посадочной;
- мини-разбор доверия.

### Образование

- диагностика набора учеников;
- чек-лист прогрева;
- карта возражений;
- аудит посадочной.

### Строительство и ремонт

- аудит доверия;
- чек-лист заявки;
- калькулятор потерь;
- диагностика упаковки услуги.

## 16. Техническая архитектура v1

Компоненты:

```text
Сайт
Telegram-бот
База заявок
ИИ-ядро
События аналитики
Платёжный контур optional
Карточка владельца
```

v1 может быть без оплаты, если продукт только сайт.

Но для продукта “Сайт с ИИ-автоворонкой” оплата трипвайера должна быть предусмотрена как v2-ready архитектура.

## 17. QA ИИ-автоворонки

Проверить:

- вход с сайта работает;
- deep-link передаёт source;
- лид-магнит выдан;
- welcome-цепочка стартует;
- ИИ задаёт релевантные вопросы;
- сегмент определяется;
- lead score считается;
- карточка владельцу отправляется;
- трипвайер предлагается;
- оплата работает, если включена;
- пользователь может запросить человека;
- пользователь может остановить сообщения;
- события аналитики фиксируются;
- sensitive data не уходит в аналитику;
- risky claims отсутствуют.

## 18. Роль в фабрике

Без ИИ-автоворонки сайт остаётся обычным сайтом.

Для продуктовой линейки Делаем Диджитал ИИ-автоворонка является усилителем конверсии и дифференциатором.

Фабрика должна производить не только страницу, а связку:

```text
Сайт -> Telegram-бот -> лид-магнит -> ИИ-прогрев -> трипвайер -> заявка -> аналитика
```
