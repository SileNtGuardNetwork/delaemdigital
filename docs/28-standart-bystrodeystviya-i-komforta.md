# Стандарт быстродействия и комфорта

Статус: v1 foundation
Назначение: зафиксировать требования к скорости, отзывчивости, адаптиву и пользовательскому комфорту premium ClientFlow-сайтов.

## 1. Главная позиция

Премиальный сайт не может быть медленным.

Если сайт красивый, но тормозит, прыгает, плохо читается на телефоне или мешает оставить заявку — он не соответствует стандарту фабрики.

Быстродействие и комфорт — часть конверсии.

## 2. Что значит комфортный сайт

Комфортный сайт:

- быстро открывается;
- не дёргается при загрузке;
- не перегружает глаза;
- читабелен на mobile;
- CTA виден рано;
- форма удобна;
- motion не мешает;
- cookie banner не блокирует действие;
- нет горизонтального скролла;
- нет тяжёлых эффектов без смысла;
- понятно, что происходит после клика.

## 3. Core Web Vitals

Ориентиры:

```text
LCP: до 2.5 сек
CLS: до 0.1
INP: до 200 мс
```

Для premium-сайтов стремиться лучше:

```text
LCP: 1.8-2.2 сек
CLS: ниже 0.05
INP: ниже 150 мс
```

## 4. Mobile-first стандарт

Проверочные ширины:

```text
360px
390px
430px
768px
1024px
1440px
```

Обязательно проверить:

- H1;
- CTA;
- форму;
- меню;
- cookie banner;
- Telegram CTA;
- длинные карточки;
- схемы ClientFlow;
- блок ИИ-автоворонки;
- pricing / product cards;
- FAQ.

## 5. Первый экран

Первый экран должен быстро показать:

- оффер;
- для кого;
- чем отличается;
- CTA;
- мягкий вход в лид-магнит или ИИ-автоворонку.

Нельзя:

- задерживать отображение hero тяжёлой анимацией;
- грузить слишком большое изображение;
- скрывать CTA ниже первого экрана;
- делать визуал важнее текста;
- делать mobile hero слишком высоким без действия.

## 6. Изображения

Правила:

- WebP/AVIF где уместно;
- hero image оптимизировать отдельно;
- priority только для критичного hero image;
- lazy loading для вторичных изображений;
- не загружать desktop image на mobile без необходимости;
- указывать размеры;
- не использовать изображения как текст;
- не использовать 5MB+ изображения в production.

## 7. Шрифты

Правила:

- использовать `next/font`;
- subsets: cyrillic и latin при необходимости;
- display swap;
- не подключать лишние начертания;
- не использовать слишком много font weights;
- контролировать CLS из-за шрифтов.

## 8. JavaScript budget

Сайт под трафик не должен быть перегружен JS.

Правила:

- не подключать тяжёлые библиотеки без необходимости;
- не делать client component без причины;
- интерактив только там, где нужен;
- серверные компоненты использовать по умолчанию;
- lazy-load тяжёлые элементы;
- analytics не должны блокировать загрузку;
- сторонние скрипты подключать осознанно.

## 9. Motion budget

Motion должен быть лёгким.

Правила:

- animating transform / opacity предпочтительно;
- не анимировать layout properties;
- не использовать heavy parallax;
- не делать scroll hijacking;
- не запускать десятки постоянных loop-анимаций;
- учитывать prefers-reduced-motion;
- motion не должен ухудшать LCP/INP.

## 10. Формы

Форма должна быть быстрой и понятной.

Проверить:

- поля легко нажимаются на mobile;
- клавиатура не ломает UX;
- submit button виден;
- loading state есть;
- success/failure state есть;
- ошибки понятные;
- введённые данные не теряются при ошибке;
- consent читаем;
- Telegram fallback есть.

## 11. Cookie banner

Cookie banner не должен блокировать основное действие.

Правила:

- не перекрывает CTA;
- mobile-friendly;
- короткий текст;
- понятные кнопки;
- ссылка на cookie policy;
- не ломает CLS;
- учитывает analytics consent.

## 12. Accessibility и удобство

Минимум:

- видимый focus state;
- label для формы;
- aria-invalid для ошибок;
- достаточный контраст;
- кнопки не меньше комфортного размера;
- alt для смысловых изображений;
- декоративные изображения с пустым alt;
- логичная структура H1-H3;
- keyboard navigation;
- prefers-reduced-motion.

## 13. Сторонние скрипты

Сторонние скрипты подключать только если они нужны:

- аналитика;
- пиксели;
- платежи;
- виджеты;
- карты;
- чат.

Правила:

- не подключать всё сразу;
- измерять impact;
- откладывать загрузку;
- не блокировать main thread;
- отключать лишнее на preview.

## 15. Smoke tests

Минимальные проверки:

- главная открывается;
- legal pages открываются;
- форма видна;
- форма валидирует;
- success/failure state работает;
- Telegram CTA работает;
- cookie banner работает;
- нет горизонтального overflow;
- нет критичных console errors;
- mobile viewport не ломается.

## 16. QA по устройствам

Проверить вручную:

- iPhone small;
- iPhone standard;
- Android standard;
- tablet;
- desktop 1440;
- desktop wide.

Особое внимание:

- hero;
- форма;
- ИИ-автоворонка;
- pricing;
- FAQ;
- footer;
- cookie banner.

Статусы:

```text

```

Критичные блокеры:

- mobile broken;
- horizontal overflow;
- CTA не виден;
- форма неудобна;

- CLS заметный;
- motion тормозит;
- cookie banner блокирует действие;
- важный контент client-only без причины;
- heavy images.

## 20. Решение

Скорость и комфорт являются частью premium-качества.

Сайт не может считаться готовым, если он красивый, но тяжёлый, неудобный или медленный.
