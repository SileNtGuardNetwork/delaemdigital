# Словарь и нейминг фабрики

Статус: v1 foundation
Назначение: зафиксировать единый язык Делаем Диджитал для документов, сайта, GPT Business, Cursor Pro+ и клиентских материалов.

## 1. Главная позиция

Фабрика должна говорить на русском языке.

Английские термины допустимы только там, где они являются техническими, продуктовыми или уже закреплёнными:

```text
ClientFlow
ClientFlow System
Digital
Next.js
Supabase
Vercel
GitHub
PostHog
Telegram
SEO
UTM
API
CRM
QA
```

Всё остальное по возможности русифицируется.

## 2. Принцип нейминга

Названия должны быть:

- понятными;
- деловыми;
- не инфобизнесовыми;
- не слишком техническими для клиента;
- точными для производства;
- едиными во всех документах.

Запрещено:

- мешать русский и английский без причины;
- использовать модные слова ради эффекта;
- называть обычный процесс “революционным”;
- использовать дешёвые названия вроде “воронка на миллион”;
- плодить синонимы для одной сущности.

## 3. Основные продуктовые названия

### Делаем Сайт

Базовый продукт.

Смысл:

```text
премиальный ClientFlow-сайт под заявки, SEO, трафик, аналитику и ИИ-автоворонку
```

### Делаем Трафик

Расширение продукта.

Смысл:

```text
сайт + рекламный маршрут + аналитика + оптимизация заявок
```

### Делаем Систему

Расширенный продукт.

Смысл:

```text
сайт + ИИ-автоворонка + Telegram-бот + аналитика + обработка лидов + улучшения
```

### ClientFlow System

Авторская технология Делаем Диджитал.

Смысл:

```text
управляемый маршрут от первого касания до заявки, обработки, аналитики и улучшения
```

## 8. Тональность Делаем Диджитал

Тон:

- спокойный;
- уверенный;
- системный;
- взрослый;
- без истерики;
- без дешёвых обещаний;
- без “мы лучшие на рынке”;
- без инфобизнес-подачи.

Хорошо:

```text
Собираем сайт как управляемый маршрут от первого касания до заявки.
```

Плохо:

```text
Мы сделаем вам взрывные продажи уже завтра.
```

## 9. Формулировки оффера

Базовая формула:

```text
Сайт с ИИ-автоворонкой под заявки, SEO и трафик по технологии ClientFlow System.
```

Допустимые варианты:

```text
Премиальный сайт с ИИ-автоворонкой, готовый к трафику, аналитике и заявкам.
```

```text
Собираем не просто сайт, а маршрут: от первого касания до заявки, прогрева и аналитики.
```

```text
Делаем сайт, Telegram-бота, лид-магнит, аналитику и маршрут заявки в одной системе.
```

## 10. Осторожные формулировки

Не писать:

```text
гарантируем заявки
gарантируем продажи
100% результат
лучший сайт на рынке
автоматически принесёт клиентов
```

Писать:

```text
готовим сайт к приёму трафика
собираем измеримый маршрут заявки
помогаем увидеть слабые места воронки
настраиваем систему для обработки и анализа заявок
```

## 14. Термины для Cursor

В заданиях Cursor можно использовать технический язык, но постановка должна быть русской.

Пример:

```text
Реализуй компонент LeadForm по стандарту формы и заявки. Используй TypeScript, Zod, server route, success/failure states и analytics events.
```

## 15. Правило для GPT Business

GPT Business должен:

- использовать русский язык по умолчанию;
- не плодить новые названия без необходимости;
- не менять продуктовые названия самовольно;
- сохранять ClientFlow и ClientFlow System;
- делать акцент на дизайн, конверсию, ИИ-автоворонку, SEO, скорость и качество.

## 16. Решение

Фабрика говорит по-русски, но использует нужные профессиональные термины.

Единый язык нужен для того, чтобы:

- не расплывалось позиционирование;
- Cursor получал понятные задания;
- GPT Business держал контекст;
- клиент видел зрелую систему;
- Делаем Диджитал выглядел как premium-производство, а не как набор случайных услуг.
