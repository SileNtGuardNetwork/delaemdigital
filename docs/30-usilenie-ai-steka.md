# Усиление AI-стека

Статус: v1 foundation
Назначение: настроить GPT Business, Claude Max и Cursor Pro+ как единую производственную систему для фабрики сайтов.

## 1. Главная позиция

Качество сайтов должно держаться не на вдохновении и не на одном чате, а на системе:

```text
GPT Business -> архитектура, стратегия, контроль качества
Claude Max -> длинные тексты, сценарии, редактура, Telegram-цепочки
Cursor Pro+ -> код, верстка, интеграции, рефакторинг, проверки
```

Инструменты не работают отдельно. Каждый инструмент получает свою роль, свои входные данные, свои ограничения и свой формат результата.

## 2. Цель настройки

Нам нужен не “помощник для сайта”, а производственный контур:

```text
Диагностика -> оффер -> конверсионный прототип -> copy-doc -> дизайн -> код -> интеграции -> QA -> запуск -> кейс
```

AI-стек должен помогать делать:

1. Конверсионные прототипы.
2. Качественный продающий копирайтинг.
3. Современный premium-дизайн.
4. Next.js-верстку высокого уровня.
5. ИИ-автоворонки в Telegram.
6. Интеграции лидов, аналитики и оплат.
7. SEO под ключ.
8. Контроль качества до запуска.

## 3. Роль GPT Business

GPT Business — главный архитектор и контролёр фабрики.

Использовать для:

- стратегии продукта;
- архитектуры фабрики;
- анализа ниши;
- создания стандартов;
- проектирования прототипа;
- проверки оффера;
- ревью структуры сайта;
- ревью дизайна;
- ревью кода на уровне архитектуры;
- QA Gate;
- анализа конкурентов;
- итогового контроля качества.

Не использовать как единственный инструмент для:

- финальной верстки без Cursor;
- длинных Telegram-цепочек без редакторского прохода;
- production-деплоя;
- самостоятельных правок секретов.

## 4. Роль Claude Max

Claude Max — сильный редактор длинных смысловых материалов.

Использовать для:

- глубокого copy-doc;
- длинных сценариев Telegram-бота;
- welcome-цепочек;
- прогрева;
- объясняющих текстов;
- редакторской шлифовки;
- сценариев лид-магнитов;
- упаковки трипвайеров;
- юридически осторожных формулировок.

Правило:

Claude не должен менять архитектуру фабрики без сверки с документами репозитория.

## 5. Роль Cursor Pro+

Cursor Pro+ — инженерный исполнитель.

Использовать для:

- Next.js app;
- TypeScript;
- Tailwind;
- компонентной системы;
- форм;
- серверных route handlers;
- Supabase;
- Telegram API;
- analytics adapters;
- SEO metadata;
- исправления ошибок;
- рефакторинга;
- локальных проверок.

Cursor не должен:

- менять оффер без approval;
- придумывать структуру сайта с нуля;
- удалять стандарты;
- трогать secrets;
- деплоить production без команды;
- заменять premium-дизайн generic-компонентами.

### Шаг 1. GPT Business

- трипвайер;

### Шаг 3. GPT Business

Проверяет:

- конверсию;
- структуру;
- риски;
- соответствие ClientFlow System.

Выход:

- адаптив;

## 10. Критерий топовой конфигурации

Конфигурация считается сильной, если:

1. Все инструменты читают одни и те же стандарты.
2. Cursor не пишет код без прототипа.
3. Claude не пишет цепочку без оффера и сегментации.
4. GPT не принимает результат без QA Gate.
5. У каждого этапа есть вход, выход и критерий готовности.
6. Любой сайт проходит через конверсию, дизайн, SEO, speed, лиды, аналитику и ИИ-воронку.

## 11. Ошибка, которой нельзя допустить

Нельзя строить фабрику как набор “умных промптов”.

Нужно строить фабрику как производственную систему:

```text
Стандарты -> агенты -> прототип -> дизайн -> код -> интеграции -> QA -> улучшение
```

## 12. Следующее действие

Создать Cursor rules и затем продолжить стандарты качества:

- дизайн-система premium-сайтов;
- motion и premium-эффекты;
- SEO под ключ;
- готовность к трафику;
- быстродействие и комфорт;
- словарь и нейминг фабрики.
