# План первого сайта Делаем Диджитал

Статус: v1 production plan
Назначение: зафиксировать, когда и как фабрика начнёт первый production-сайт Делаем Диджитал.

## 1. Главный принцип

Первый сайт Делаем Диджитал нельзя делать как ручной разовый лендинг.

Он должен быть первым доказательством фабрики:

```text
factory-template
→ spec-driven workflow
→ premium design system
→ lead-layer
→ AI funnel
→ QA Gate
→ launch
→ case
```

## 2. Когда начинать

Начинать production-разработку сайта можно только после:

```text
v0.2 merged
lead-layer dev QA passed
plan v0.3 accepted
premium design/media standard accepted
```

Можно заранее готовить:

- discovery;
- spec;
- conversion blueprint;
- copy-doc;
- design direction;
- photo/motion requirements.

## 3. Цель сайта

Сайт должен продавать не “разработку сайтов”, а продукт:

```text
Фабрика сайтов с ИИ-автоворонками по технологии ClientFlow System,
готовые к приёму трафика, заявок и SEO под ключ.
```

## 4. Основной оффер

Черновой оффер:

```text
Собираем premium-сайт с ИИ-автоворонкой под заявки, SEO и трафик — от оффера и структуры до формы, базы лидов, Telegram-цепочки и аналитики.
```

## 5. Целевая аудитория

Основные сегменты:

- эксперты;
- консультанты;
- образовательные проекты;
- агентства и сервисные компании;
- B2B-услуги;
- локальный бизнес с дорогой заявкой;
- предприниматели, которые уже пробовали сайт/рекламу, но не видят системы.

## 6. Структура сайта v1

### 1. Hero

Цель: за 5 секунд объяснить продукт.

Состав:

- H1;
- subtitle;
- primary CTA;
- secondary CTA;
- 3 proof bullets;
- premium visual / system map / founder-led visual.

### 2. Где бизнес теряет клиентов

Цель: показать проблему не в “сайте”, а в пути клиента.

### 3. ClientFlow System

Цель: объяснить авторскую технологию.

### 4. Что входит в сайт под ключ

Цель: показать, что это не лендинг, а система.

Блоки:

- оффер;
- структура;
- копирайтинг;
- дизайн;
- верстка;
- форма;
- база лидов;
- Telegram;
- аналитика;
- SEO;
- legal;
- QA.

### 5. ИИ-автоворонка

Цель: показать ключевую дифференциацию.

Состав:

- Telegram-бот;
- лид-магнит;
- welcome-chain;
- ИИ-ядро;
- сегментация;
- трипвайер;
- owner notification.

### 6. Для кого

Цель: сегментировать аудиторию.

### 7. Процесс работы

Цель: снять страх хаоса.

Этапы:

```text
Диагностика
→ Прототип
→ Copy-doc
→ Дизайн
→ Верстка
→ Интеграции
→ ИИ-воронка
→ QA
→ Запуск
```

### 8. Пакеты

Цель: монетизация.

Пакеты позже уточнить.

### 9. Кейсы / внутренний кейс

На старте:

```text
как мы собрали собственную фабрику
```

### 10. FAQ

Цель: закрыть возражения.

### 11. Финальный CTA

Цель: заявка / диагностика.

## 7. Lead magnet

Первый лид-магнит:

```text
Диагностика: где ваш сайт или воронка теряет клиентов
```

Формат:

- Telegram mini-diagnostic;
- 7 точек ClientFlow;
- итоговый score;
- рекомендации;
- предложение аудита / созвона / трипвайера.

## 8. ИИ-автоворонка для сайта

Минимальный путь:

```text
CTA на сайте
→ Telegram bot
→ лид-магнит
→ 3-5 вопросов
→ сегмент
→ lead score
→ персональный next step
→ owner card
→ tripwire / диагностика
```

## 9. Визуальное направление

Базовое направление:

```text
Dark Technology Premium + Founder/Operator trust
```

Код:

- graphite / obsidian;
- copper accent;
- steel-blue light;
- cinematic office / system map;
- route-line ClientFlow;
- premium typography;
- restrained motion.

## 10. Фото / видео

Для первого сайта желательно использовать:

- портрет основателя;
- dark office hero;
- system map overlay;
- optional cinematic video loop;
- mobile fallback.

Правило:

```text
H1, оффер и CTA только HTML, не внутри видео.
```

## 11. Техническая база

Использовать:

- `apps/factory-template`;
- v0.2 lead-layer;
- v0.3 AI funnel после реализации;
- future Playwright QA Gate;
- SEO / traffic standard;
- premium design/media system.

## 12. Production gates

Нельзя запускать сайт без:

- build;
- typecheck;
- lint;
- manual mobile QA;
- form QA;
- Supabase QA;
- Telegram QA;
- analytics QA;
- SEO QA;
- legal review;
- design QA;
- launch checklist.

## 13. Сроки

После merge v0.2:

```text
1-2 дня: spec + blueprint + copy-doc
2-4 дня: premium design + hero/media
2-4 дня: implementation
2-4 дня: AI funnel integration
1-2 дня: QA + deploy
```

Оценка:

```text
первый рабочий сайт без полной ИИ-воронки: 5-7 рабочих дней
сайт с ИИ-автоворонкой: 8-14 рабочих дней
production-ready сайт: 10-16 рабочих дней
```

## 14. Критерий успеха

Первый сайт успешен, если:

- выглядит premium;
- объясняет фабрику за 5 секунд;
- имеет сильный lead magnet;
- принимает реальные заявки;
- запускает ИИ-воронку;
- сохраняет лидов;
- уведомляет владельца;
- готов к SEO и трафику;
- проходит QA Gate;
- может стать публичным кейсом.
