# Delaem Digital DESIGN.md

Status: Open Design design system for spike use.
Version: 1.0

## Brand thesis

Delaem Digital builds ClientFlow Systems for business owners and experts.

The visual language must communicate:

- strategic control;
- premium digital production;
- founder-led responsibility;
- system thinking;
- calm authority;
- measurable client flow;
- no visual noise.

This is not a generic web studio and not a startup landing page.

## Core offer context

Delaem Digital does not sell isolated websites, bots, traffic campaigns or AI wrappers.

It builds a connected system:

```text
Brand -> Website -> Traffic -> Lead -> CRM / Telegram -> Processing -> Analytics -> Improvement
```

The website must make this system visible.

## Visual style

Primary style:

```text
premium dark B2B
founder-led editorial
architectural system map
controlled command interface
```

Mood:

- precise;
- expensive;
- restrained;
- technical but human;
- calm but confident;
- structured;
- high-ticket.

## Color system

Use dark premium base:

```text
canvas: #07070d
surface: #0f0f1a
surface_alt: #121422
border: #1f2433
border_soft: rgba(255,255,255,0.08)
text: #ededf3
text_muted: #9aa3b2
text_dim: #6f7685
copper: #c97a50
steel_blue: #5b8cff
deep_blue: #5266eb
success: #48c78e
warning: #d99a45
danger: #ef6b6b
```

Accent discipline:

- copper = commercial action / primary CTA;
- steel blue = system / data / architecture;
- green = status/success only;
- red = warning/error only.

Do not use neon gradients, purple SaaS gradients, rainbow effects or cyberpunk glows.

## Typography

Typography must carry the premium feeling.

Rules:

- large controlled display heading;
- strong line breaks;
- concise subheadline;
- readable body copy;
- no tiny critical text;
- no overuse of uppercase;
- no more than three text tones in one screen.

Preferred feel:

- editorial;
- architectural;
- system documentation;
- high-end B2B.

## Layout

Use disciplined layout:

```text
max-width: 1200-1360px
12-column grid or strict flex grid
large section spacing
clear content hierarchy
CTA visible without hunting
```

Hero layout should usually be hybrid:

```text
left: founder/offer/copy/CTA
right: ClientFlow architecture visual or premium command interface
```

Mobile:

- text first;
- CTA early;
- visual becomes secondary;
- no horizontal overflow;
- no dense dashboards.

## Hero concepts

Generate only these families for the first spike:

### Concept A — Founder Architect

Founder-led, editorial, personal trust, calm authority.

Visual anchor:

- founder portrait area or editorial silhouette;
- compact proof markers;
- dark premium surface;
- subtle system overlay.

### Concept B — ClientFlow System Map

The system is the hero.

Visual anchor:

- connected route modules;
- traffic -> website -> lead -> Telegram/CRM -> analytics;
- clean lines;
- no fake dashboards.

### Concept C — Premium Command Interface

Operational control feel.

Visual anchor:

- real-looking command panel;
- lead score card;
- ClientFlow route;
- analytics/status states;
- restrained depth.

## Required hero copy placeholders

Use or preserve this meaning:

```text
Kicker: Vitaly Timoshenko / Delaem Digital
Headline: Build a managed ClientFlow system for your business
Subheadline: Website, lead capture, Telegram/CRM routing, analytics and AI qualification connected into one client acquisition system.
Primary CTA: Get ClientFlow audit
Secondary CTA: See how the system works
Trust marker: First we build the system for ourselves. Then we apply it to clients.
```

If writing in Russian, use a premium, direct, non-hype tone.

## Section patterns

Preferred sections:

- Hero — Direct Outcome;
- Problem — Cost of Chaos;
- Solution — ClientFlow Map;
- Offer Cards;
- Trust Stack;
- Internal Case / Build in Public;
- Case Preview;
- Diagnostic CTA;
- FAQ;
- Final CTA.

Every section must answer a user question and move the user toward action.

## Motion

Motion must be restrained.

Allowed:

- line-by-line hero reveal;
- soft section reveal;
- route-line draw for ClientFlow Map;
- proof card stagger;
- subtle CTA hover.

Forbidden:

- scroll hijacking;
- aggressive parallax;
- floating random cards;
- continuous loops;
- neon glow animation;
- motion that blocks forms or readability.

## Forbidden visual patterns

Do not create:

- generic SaaS landing;
- fake dashboards with fake metrics;
- random glassmorphism cards;
- cyberpunk visuals;
- crypto/web3 aesthetics;
- startup purple gradients;
- excessive 3D blobs;
- noisy icon clouds;
- stock-photo trust replacement;
- AI raster text.

## Output requirement

When generating a prototype, output a single strong visual direction, not a collage of ideas.

The design must be implementable in:

```text
Next.js + TypeScript + Tailwind + shadcn/ui + GSAP
```

Use real text layers, real layout, real components and responsive structure.

## Quality target

The prototype should be capable of reaching 85/100 in DD Visual QA after implementation.

Priority order:

1. Clear offer.
2. Premium typography.
3. Strong layout.
4. Brand fit.
5. Conversion path.
6. Mobile feasibility.
7. Motion potential.
