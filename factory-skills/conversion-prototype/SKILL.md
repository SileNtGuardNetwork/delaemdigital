# Conversion Prototype Skill

## Когда использовать

Использовать, когда нужно создать структуру будущего сайта:

- первый экран;
- порядок секций;
- CTA;
- логика убеждения;
- возражения;
- lead magnet entry;
- финальный CTA.

## Вход

- ниша;
- продукт / услуга;
- целевая аудитория;
- оффер;
- боли;
- желаемое действие;
- ограничения;
- traffic source if known.

## Процесс

1. Определить главный conversion job сайта.
2. Сформулировать H1 logic.
3. Построить секционную карту.
4. Назначить цель каждой секции.
5. Расставить CTA.
6. Добавить возражения и trust-блоки.
7. Определить место lead magnet.
8. Сформировать финальный CTA.

## Выход

```text
conversion blueprint
section map
CTA map
objection map
lead magnet entry
```

## QA

Прототип готов, если:

- каждый блок имеет цель;
- первый экран понятен за 5 секунд;
- CTA повторяются логично;
- есть работа с возражениями;
- есть путь к заявке / лид-магниту;
- нет случайных секций.

## Запреты

- не писать финальный дизайн;
- не начинать верстку;
- не использовать generic SaaS copy;
- не добавлять блоки без роли в конверсии.
